# Second Dataset Selection for FAIR-ML-CYBER

Date: 2026-06-17

## Decision Summary

Primary recommended second dataset:

**CIC-UNSW-NB15 Augmented Dataset**

Official source:

https://www.unb.ca/cic/datasets/cic-unsw-nb15.html

This is the best immediate candidate because it is an external underlying dataset derived from UNSW-NB15, while still using CICFlowMeter-style flow extraction. This gives a realistic chance of integrating it into the current FAIR-ML-CYBER pipeline without fabricating data or rewriting the entire feature layer.

## Non-Negotiable Rule

No generated/RAG-created records will be used as if they were real traffic. If synthetic data is ever used, it must be explicitly labelled as synthetic and excluded from empirical claims about real datasets.

RAG can be used for:

- extracting dataset documentation;
- mapping feature names;
- building data dictionaries;
- generating preprocessing code;
- checking citation and licensing notes.

RAG must not be used to fabricate hidden validation data.

## Source Notes

### CIC-UNSW-NB15 Augmented Dataset

The UNB page states that the dataset was generated by using CICFlowMeter to extract flows from UNSW-NB15 packet traffic and label them using UNSW-NB15 ground truth. It lists four dataset files:

- `CICFlowMeter_out.csv`
- `Data.csv`
- `Label.csv`
- `Readme.txt`

The page also reports category counts for benign and attack classes and provides a dataset download link.

Use in FAIR-ML-CYBER:

- primary second-dataset validation;
- test whether the same stress-test conclusions hold outside the local CICIDS2017-like files;
- align flow features with current CICFlowMeter-style feature selection where possible;
- cite both the augmented dataset and the original UNSW-NB15 provenance.

### CSE-CIC-IDS2018

The official UNB page describes seven attack scenarios: brute-force, Heartbleed, Botnet, DoS, DDoS, Web attacks and infiltration. It includes captured network traffic and system logs, with 80 features extracted using CICFlowMeter-V3.

Use in FAIR-ML-CYBER:

- backup second dataset if CIC-UNSW-NB15 integration is blocked;
- useful for same-family temporal/scenario validation;
- not ideal as the only second dataset if reviewers demand stronger independence.

### UNSW-NB15 Original

The official UNSW page states that UNSW-NB15 was created using IXIA PerfectStorm in the UNSW Canberra Cyber Range Lab. It includes nine attack types and 2,540,044 records stored in four CSV files, with a separate predefined training set of 175,341 records and testing set of 82,332 records.

Use in FAIR-ML-CYBER:

- stronger independent external validation;
- requires feature harmonization because the schema differs from CICFlowMeter.

### TON_IoT

The UNSW page describes ToN-IoT as a collection of IoT/IIoT telemetry, OS logs and network traffic datasets, including processed CSV datasets and train/test datasets. Academic use is granted, with citation requirements.

Use in FAIR-ML-CYBER:

- possible external domain-shift appendix;
- high value but larger scope change.

### UQ NetFlow Collections

The University of Queensland NIDS datasets page provides NetFlow V1/V2/V3 datasets in CSV format. It also includes CICFlowMeter datasets. NetFlow V3 includes temporal features and multiple standardized dataset variants.

Use in FAIR-ML-CYBER:

- best long-term route for a standardized cross-dataset article;
- useful if the paper evolves from CICFlowMeter stress testing to standardized NetFlow transferability.

## Proposed Experimental Path

### Minimal Q1 Repair

1. Download CIC-UNSW-NB15.
2. Audit files, labels and feature schema.
3. Map shared CICFlowMeter-compatible features to `deployment_safe`.
4. Run binary baseline on the second dataset.
5. Report cross-dataset consistency: random split vs stress-test gap, calibration, rare classes if present.

### Stronger Q1 Repair

1. Add CIC-UNSW-NB15 as second dataset.
2. Add one standardized UQ NetFlow dataset as a robustness appendix.
3. Add an explicit feature-harmonization layer.
4. Report which conclusions persist across schemas and which are CICFlowMeter-specific.

## Manuscript Rule

The manuscript must cite the second dataset source and describe preprocessing honestly. It should never say or imply that generated data is real observed network traffic.
